what is the next step in creating the website

Great question. The next step is to move from a visual prototype to a usable learning product.

Recommended immediate next step (do this first)

1. Add structured lesson content tied to each animation

Right now, the site already has:

  • a landing/hero,

  • three quick rules,

  • three encounter modes,

  • and signal visuals.
    All of that is in place and working as a shell.

The next high-impact step is to add a “Scenario → Rule → Action” panel for each encounter mode:

  • Scenario: what vessels are doing,

  • Rule reference: COLREG rule number/title,

  • Required action: who gives way, expected maneuver,

  • Common mistakes: what not to do.

This makes the animation instructional instead of just decorative. The current simulator already switches modes and text, so this is a natural extension of your existing structure.

After that, do these in order

  1. Improve simulator realism

    • show headings, CPA/TCPA hints, and relative bearing lines in the canvas.

  2. Add accessibility + controls

    • pause/play animation, keyboard navigation for mode buttons, reduced-motion support in CSS.

  3. Add credibility layer

    • include references/links to official COLREG text and a “training aid disclaimer” near each module (you already have a footer disclaimer).

  4. Publish

    • deploy as static site (GitHub Pages / Netlify / Vercel) and test on mobile.

Why this is the right next step

Your current codebase already provides strong UI/animation foundations:

  • content sections and layout are present,

  • styling/animation system is already built,

  • encounter switching logic already exists.

So the biggest value now is instructional depth, not more visual polish first.
